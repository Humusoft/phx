phx.assembly.import

Robot model imported from a URDF file

Function  phx.assembly.import  |  Package: phx.assembly

Description

phx.assembly.import reads a robot description from a URDF (Unified Robot Description Format) file and creates a phx.Body for every link and a joint for every revolute, continuous, prismatic or fixed joint of the robot. Despite the name of the format, URDF files describe more than robots — ragdolls, vehicles, furniture and single-body props from public model libraries import the same way. The assembled model is placed in its zero (home) configuration with the base (root link frame) at the world origin, unless a pose is given by the Position and Orientation or EulerAngles options.

Creation

Syntax

bodies = phx.assembly.import(file) [bodies,joints] = phx.assembly.import(file) [___] = phx.assembly.import(___,Name,Value) [___] = phx.assembly.import(ax,___)

Description

bodies = phx.assembly.import(file) creates the bodies in the current axes and returns them in a struct whose field names are the link names (made valid MATLAB identifiers; the original URDF name is preserved in the Name property). [bodies,joints] = phx.assembly.import(file) also returns all created joints in a struct keyed by the joint names.

[___] = phx.assembly.import(ax,___) draws into the given axes instead of the current axes; an empty target ([]) creates the bodies without graphics for headless simulations.

Options

OptionDefaultDescription
Position[0 0 0]World position of the base (root link frame).
Orientationeye(3)World rotation of the base as a 3×3 rotation matrix. Same convention as phx.Body.Orientation.
EulerAngles[0 0 0]World rotation of the base as Euler angles (z→y→x order), an alternative to Orientation.
MeshPath""Root folder used to resolve package:// URIs and relative file names of mesh geometries. Meshes are always searched relative to the folder of the URDF file as well.

URDF to PHX mapping

Every link becomes a phx.Body with the body frame at the origin of the link's first visual geometry (PHX shapes are always centred at the body origin). Geometries map to Box, Cylinder, Sphere, Capsule (a common vendor extension) and STL / OBJ meshes with the convex collision envelope. Joints of type revolute and continuous become phx.RevoluteJoint, prismatic joints become phx.PrismaticJoint (sliding along the joint axis) and fixed joints become phx.FixedJoint; planar and floating joints are substituted by a fixed joint with a warning. Joint limits are ignored and all joints are passive — no motors are created. Masses come from the inertial elements; only the diagonal of the inertia tensor expressed in body axes is used.

Examples

Import a robot and anchor its base

clf; view(3); axis equal; grid on; camlight headlight;

[bodies, joints] = phx.assembly.import("robot.urdf");
bodies.base.Type = "static";

sim = phx.Simulation(bodies);
sim.step(1, 500, 5);
delete(sim);

Place a ragdoll leaning over an edge

[bodies, joints] = phx.assembly.import("human.urdf", ...
    "Position", [-0.16 0 2.6], "EulerAngles", [0 0.66 0]);
