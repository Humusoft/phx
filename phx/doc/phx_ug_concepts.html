How PHX Thinks — Core Concepts

The handful of object types every scene is built from

Every scene is built from just a handful of object types. Once you understand them, the rest is only a matter of combining them.

ObjectWhat it represents
phx.Simulation The whole scene and its “engine”. Holds time and gravity, and steps the computation.
phx.Body A rigid body — it has a position, mass, velocity and a shape. The basic building block.
phx.shape.* The geometry a body “wears” — box, sphere, cylinder, STL model … See Bodies and shapes.
phx.*Joint A connection of two bodies with restricted motion (hinge, ball joint, gear). See Joints, springs and ropes.
Forces and fields Spring, Rope, Thruster, Resistance, Buoyancy, Monopole, Dipole — springs, actuators, environmental drag, floating, gravitational/magnetic interaction.
Assemblies phx.assembly.* — prefab builders that create whole groups of parts at once (an arena, a jointed chain, scattered bodies, a URDF robot). See Prefab assemblies.
Observers Logger (data logging), Trace (motion trail), Measure (distances and velocities). See Measuring and logging.
Automation Script (prescribed value profiles over time), Function (custom computation/control at every step). See Automation and control.

Three steps of every scene

  1. Build — create the bodies and their links (in any MATLAB axes).
  2. Run — create a phx.Simulation and call step.
  3. Evaluate — read body properties or data from a Logger.
% the skeleton of every scene
clf; view(3); axis equal; grid on; camlight headlight;   % scene
phx.Body(...);                                           % bodies
phx.Body(...);
sim = phx.Simulation;                                    % engine
sim.step(interval, substeps, redrawStep);                % run
delete(sim);                                             % cleanup

Stepping — the step method

The most important simulation command:

sim.step(interval, substeps, redrawStep)
  • interval — how many seconds to simulate,
  • substeps — into how many sub-steps the interval is split (computation granularity),
  • redrawStep — how often the view is redrawn (1 = every step).
Note. A smaller step (more substeps) = a more stable but slower computation. Tightly coupled scenes — joint networks, stacks in contact — call for a finer step. Bodies can also be added to a running simulation with sim.addObjects(b).
See also phx.Simulation, phx.Body
