phx.Rope

Rope routed over pulleys

Class  phx.Rope  |  Superclass: phx.base.Object

Description

phx.Rope connects a chain of two or more bodies. The first and the last body are the rope ends; every body in between acts as an ideal frictionless pulley, so a single tension acts along the whole rope. The routing points are defined in the local spaces of the bodies by the rows of the Points matrix. A body may appear in the chain more than once with different routing points — for example the moving block of a tackle that the rope wraps twice.

The rope transfers tension only: it behaves as a spring–damper when stretched beyond its free length and goes slack otherwise. The free length is measured from the initial configuration (InitialLength) and can be changed while the simulation runs through the Displacement property — positive values pay the rope out, negative values winch it in. Displacement can be driven by a phx.Script, and Length and Force can be recorded by a phx.Logger.

Creation

Syntax

rope = phx.Rope(bodies) rope = phx.Rope(___,Name,Value)

Description

rope = phx.Rope(bodies) routes a rope through the origins of the given bodies (at least two). Set the routing with Points (one row per body).

Input Arguments

ArgumentTypeDescription
bodiesphx.Body (1×n, n≥2)Bodies forming the rope chain, in routing order.

Examples

Hang a load from a fixed anchor

anchor = phx.Body("Type", "static", "Position", [0 0 5]);
load   = phx.Body("Position", [0 0 3], "Mass", 10);

rope = phx.Rope([anchor load], "Stiffness", 5e3, "Damping", 50, ...
    "Colormap", "hot", "ColorRange", [0 200]);

sim = phx.Simulation;
sim.step(4, 800, 5);

Winch the rope in with a script

phx.Script(rope, {'Displacement', [0 2], [0 -1], 'linear', 'nearest'});
phx.Logger(rope, "Frequency", 50, "Parameters", ["Length" "Force"]);
