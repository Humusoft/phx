phx.base.Joint

Superclass for custom kinematic joints

Abstract class  phx.base.Joint  |  Superclass: phx.base.Object

Description

phx.base.Joint is the abstract superclass for kinematic constraints connecting two bodies, referred to as body A and body B. Concrete joints derive from it and define the actual constraint; this class provides the reaction force and torque feedback common to all of them through the ForceA, TorqueA, ForceB and TorqueB properties.

Properties

PropertyTypeDescription
ForceA1×3 double (read-only)Reaction force acting on body A, N.
TorqueA1×3 double (read-only)Reaction torque acting on body A, N·m.
ForceB1×3 double (read-only)Reaction force acting on body B, N.
TorqueB1×3 double (read-only)Reaction torque acting on body B, N·m.

Before the joint is initialized in a simulation, these properties return [NaN NaN NaN]. They also inherit the full phx.base.Object property set (Children, Color, Name, …).

Examples

Record joint reaction forces with a logger

A = phx.Body("Type", "static", "Position", [1 1 0]);
B = phx.Body("Position", [4 1 0]);
C = phx.Body("Position", [4 -2 0]);

phx.RevoluteJoint(A, B, "PointA", [1.5 0 0], "PointB", [-1.5 0 0]);
S = phx.SphericalJoint(B, C, "PointA", [0 -1.5 0], "PointB", [0 1.5 0]);

log = phx.Logger(S, "Frequency", 100, "Parameters", ["ForceA" "ForceB"]);

sim = phx.Simulation;
sim.step(6, 600, 1);
delete(sim);

plot(log.Time, log.getChannel(1));     % reaction force on body A
