Automation and Control: Script and Function

Inserting your own logic into the simulation loop

A scene need not just fall passively — you can insert your own logic that runs at every step. Two add-on objects exist for this.

phx.Script — a prescribed profile over time

Declarative (feedforward) control: to each property you assign either an interpolated curve (times + values), or a function of time given as a string. This is ideal for moving kinematic bodies — conveyors, arms, drive cams.

% steady rotation of a screw: EulerAngles as a function of time t
phx.Script(screw, {"EulerAngles", "[2*pi*t*0.5 0 0]"});

% prescribed path of an arm: interpolation of given points (pchip method)
phx.Script(arm, {"Position", time, position, "pchip"}, ...
                {"EulerAngles", time, angle});

phx.Function — a custom computation at every step

Inserts an arbitrary computation into the simulation loop without writing a custom class. The supplied callback is called every (sub)step with the signature fcn(obj, parents, dt, time) and acts by modifying the bound bodies directly. State between steps can be kept in obj.UserData. It is the ideal tool for feedback control, custom force laws or filters.

% a time-dependent periodic force
phx.Function(body, @(obj, p, dt, t) p{1}.applyForce([0 0 50]*sin(t)));

% a controller reading the scene state and driving several bodies (feedback)
phx.Function({ball, plate}, @(obj, p, dt, t) balanceLaw(p, Kp, Kd));
Which one when. Use phx.Script when you know the profile in advance. Use phx.Function when you need to react to the current state (read position/velocity and compute a control action). This exact feedback pattern is used by the self-balancing robot example.
