phx.Simulation

Physical simulation

Class  phx.Simulation  |  Superclass: phx.base.Object

Description

phx.Simulation is the topmost object in the hierarchy of a physical scene. It collects a set of phx.Body objects together with all of their children (joints, springs, force elements, traces, loggers…), builds the execution pipelines that advance and render them, and steps the underlying physics engine forward in time.

Adding or removing objects requires a rebuild of the simulation pipeline. This operation is done automatically, but may take a while.

Deleting a simulation object disconnects the engine, but if the visualization window is open, the model objects themselves remain and the simulation object can be recreated.

Creation

Syntax

sim = phx.Simulation sim = phx.Simulation(bodies) sim = phx.Simulation(___,Name,Value)

Description

sim = phx.Simulation creates a simulation of all bodies found in the current axes (gca) and their children.

sim = phx.Simulation(bodies) creates a simulation of the given bodies and their children. bodies may be a phx.Body array or cell array, an axes object, the name of a MAT file, or a struct of bodies.

Pass [] only through one of the explicit sources; an axes handle is the usual choice.

sim = phx.Simulation(___,Name,Value) sets properties using one or more name-value arguments, for example phx.Simulation(bodies,"Gravity",[0 0 0]).

Input Arguments

ArgumentAccepted typesDescription
bodiesphx.Body | cell | axes | char/string | struct Source of the bodies to simulate. An axes (or UIAxes) collects every body drawn into it; a char/string is treated as a MAT file to load; a struct is scanned for its phx.Body fields.

Properties

PropertyType / DefaultDescription
Gravity[0 0 -9.81] Global gravitational acceleration applied to the world, m/s².
ExcludeInvisibletrue Exclude all initially invisible objects from rendering.
HideInvalidfalse Turn off visibility for all objects that become invalid during the simulation.
EngineSettings[] Initial engine-specific settings, e.g. a phx.engine.BulletSettings object.
Timedouble (read-only) Current simulation time, in seconds. Starts at 0 and advances with each step.

Properties inherited from phx.base.Object include Children, Parents, Name and UserData.

Object Functions

FunctionDescription
stepPerform one or more simulation steps.

dispStructureDisplay the object tree of the scene.
dispPipelinesDisplay the compute and compute-and-redraw execution pipelines.
deleteDestroy the engine world and remove the simulation.

step

step(sim,interval) step(sim,interval,substeps) step(sim,interval,substeps,redrawStep)

Advances the simulation by interval seconds. With substeps the interval is divided into that many engine steps of size interval/substeps; rendering is then performed only once, on the last substep. With redrawStep the view is additionally redrawn every redrawStep-th substep. Use a value of -1 to suppress rendering entirely (headless runs). Smaller substeps improve the stability of stiff constraint networks.

addObjects

bodies = addObjects(sim,axes) bodies = addObjects(sim,fileName) bodies = addObjects(sim,bodies)

Adds bodies (and all their children) to a running simulation and rebuilds the pipelines, then returns the added bodies for further manipulation.

Examples

Step a minimal scene

clf; view(3); axis equal; grid on; camlight headlight;

phx.Body("Position", [0.6 -0.5 2]);   % dynamic body
phx.Body("Type", "static");             % ground

sim = phx.Simulation;                  % all bodies in the current axes
sim.step(1, 100, 1);                   % 1 s in 100 substeps, redraw each step

Custom gravity and engine settings

% Zero gravity, do not auto-deactivate resting bodies
sim = phx.Simulation(gca, "Gravity", [0 0 0], "EngineSettings", ...
    phx.engine.BulletSettings("AutoActivated", false));
sim.step(4, 800, -1);                  % run headless (no redraw)

Add and delete objects between steps

sim = phx.Simulation(boxes, "HideInvalid", true);
sim.step(5, 500, 5);

sim.addObjects(newBox);                % rebuilds pipelines
sim.step(10, 1000, 5);

delete(boxes(7));                      % also triggers a rebuild
sim.step(5, 500, 5);
delete(sim);                           % tear down the engine world
