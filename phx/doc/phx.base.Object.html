phx.base.Object

Superclass for custom simulable objects

Abstract class  phx.base.Object  |  Superclass: matlab.mixin.SetGetExactNames

Description

phx.base.Object is the superclass of every simulable object in PHX — bodies, joints, springs, force elements and helpers all derive from it. It implements a directed acyclic graph: each object holds cell arrays of Parents and Children, owns an hgtransform for its graphics and a handle to its engine counterpart.

Subclass this object (or one of phx.Body, phx.base.Joint, phx.base.Shape) to create your own simulable element, and implement the four abstract methods below. An empty body is acceptable when a hook is not needed.

Abstract Methods

A subclass must implement the following. They define how the object maps to the engine and how it is advanced and drawn.

MethodRole
valid = initObject(obj,world) Create or refresh the engine counterpart. Called whenever pipelines are (re)built. Return false to exclude the object from the simulation.
destroyObject(obj) Tear down the engine counterpart. Called from delete.
resolveState(cellObjs,dt,time,world) Static. Advance state each (sub)step. Receives a batch of objects of one class.
updateView(cellObjs,dt,time,world) Static. Sync graphics each render step. Receives a batch of objects of one class.
Performance. resolveState and updateView are static and receive a whole batch of same-class objects at once — this batching is the core performance idea behind the execution pipelines. Route all engine access through phx.engine.io.

Examples

Inspect the scene graph

A = phx.Body("Name", "base", "Type", "static");
B = phx.Body("Name", "arm", "Position", [2 0 0]);
phx.RevoluteJoint(A, B);

sim = phx.Simulation;
sim.dispStructure;                     % print the object tree

Propagate a property to a whole branch

sim.propagate("Visible", false);          % hide the simulation and everything under it
See also phx.Simulation, phx.Body, phx.base.Joint, phx.base.Shape
