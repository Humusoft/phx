phx.Function

Custom computation in the simulation pipeline

Class  phx.Function  |  Superclass: phx.base.Object

Description

phx.Function inserts an arbitrary computation into the simulation pipeline without writing a custom phx.base.Object subclass. It is the lightweight middle ground between phx.Script (declarative, feedforward value profiles) and a full custom class: use it to apply custom force laws, feedback control, filters or any per-step logic to one or more bodies.

The supplied callback is called at every simulation step (including substeps) with the signature

fcn(obj, parents, dt, time)

where obj is the phx.Function instance (use its UserData property to keep internal state between steps), parents is the cell array of the bound objects (same as obj.Parents) and dt, time are the substep size and the current simulation time. The callback has no return value — it acts by modifying the bound objects directly.

Creation

Syntax

f = phx.Function(parents,fcn)

Description

f = phx.Function(parents,fcn) creates the object, binds it to the given objects (array or cell array) and assigns the callback.

Input Arguments

ArgumentTypeDescription
parentsphx object array or cell arrayThe objects passed to the callback. A cell array may mix objects of different classes.
fcnfunction_handle (scalar)Callback executed each simulation step: fcn(obj, parents, dt, time).

Properties

PropertyType / DefaultDescription
Callbackfunction_handleCallback executed each simulation step: fcn(obj, parents, dt, time).

Examples

Apply a time-dependent periodic force

phx.Function(body, @(obj, p, dt, t) p{1}.applyForce([0 0 50]*sin(t)));

Feedback control reading the scene state

% a controller reading the scene state and driving several bodies
phx.Function({ball, plate}, @(obj, p, dt, t) balanceLaw(p, Kp, Kd));

sim = phx.Simulation;
sim.step(5, 1000, 2);
